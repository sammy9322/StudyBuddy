📖 Plan de Implementación Detallado — StudyBuddy

Documento: Plan de implementación completo para la aplicación StudyBuddy, basado en el análisis exhaustivo del archivo studybuddy.html.

Objetivo: Construir una aplicación web educativa que permita a estudiantes (6-15 años) fotografiar material escolar y generar resúmenes, quizzes, flashcards y mapas conceptuales usando IA (Google Gemini).

Arquitectura: SPA (Single Page Application) sin dependencias externas, frontend-only con API de Gemini directa.

1. Estructura del Proyecto y Configuración Inicial

1.1. Análisis del Código Existente

  • El archivo studybuddy.html contiene toda la aplicación en un único archivo (HTML + CSS + JS)
  • Utiliza HTML5, CSS3 y JavaScript ES6+ sin dependencias externas ni bundlers
  • Implementa una arquitectura de pestañas para navegación entre 4 secciones: Inicio, Generar, Practicar, Historial
  • Usa localStorage para persistencia de datos (configuración e historial)
  • Estado global manejado en un objeto state con: images, config, currentQuiz, quizIndex, quizScore, quizAnswered, history, genTypes

1.3. Decisión de Arquitectura NUEVO

La aplicación se mantendrá como un archivo único HTML (single-file SPA) por las siguientes razones:

  • El público objetivo (padres/tutores) necesita máxima simplicidad de despliegue
  • Se puede abrir directamente desde el sistema de archivos sin servidor
  • Sin dependencias de build tools, npm, ni bundlers
  • El CSS y JS están embebidos para facilitar distribución
⚠️ Consideración futura: Si la aplicación crece significativamente, evaluar migración a Vite con módulos ES separados (CSS, JS por componente).

2. Requisitos Funcionales por Categoría

2.1. Gestión de Configuración

  • Configuración de nombre del estudiante (input text, placeholder: "Sofía")
  • Selección de grado/nivel educativo con 3 opciones:
    • primaria_baja: 1°-3° Primaria (6-9 años)
    • primaria_alta: 4°-6° Primaria (9-12 años) — default
    • secundaria: Secundaria (12-15 años)
  • Configuración de API Key de Gemini con campo tipo password y toggle de visibilidad (👁)
  • Enlace directo a Google AI Studio para obtener key
  • Persistencia en localStorage bajo la clave sb_config (JSON: {name, grade, apiKey})
  • Mensaje informativo: "Tu API Key se guarda solo en tu navegador. Nunca se envía a ningún servidor nuestro."

2.2. Captura y Gestión de Imágenes MEJORADO

  • Subida desde archivos: Input tipo file con atributo accept="image/*" y multiple
  • Drag & Drop:
    • Zona de drop con handlers: ondrop, ondragover, ondragleave
    • Feedback visual con clase .drag-over que cambia borde y fondo
    • Filtro de archivos dropeados: solo file.type.startsWith('image/')
  • Cámara del dispositivo:
    • Acceso via navigator.mediaDevices.getUserMedia con facingMode: 'environment'
    • Stream de video en elemento <video> con autoplay playsinline
    • Captura de foto: dibujar frame en <canvas> y convertir a JPEG 92% calidad
    • Cierre de cámara con stop de tracks del stream
  • Vista previa:
    • Grid responsivo con grid-template-columns: repeat(auto-fill, minmax(110px, 1fr))
    • Cada imagen muestra: preview, botón eliminar (✕), número de página
    • Almacenamiento en state.images[] como objetos { file, dataUrl, base64, mimeType }
  • Formatos soportados: JPG, PNG, WEBP

2.3. Generación de Contenido con IA MEJORADO

  • Selección múltiple de tipos: Sistema de pills con Set para seleccionar tipos de generación. Mínimo 1 tipo debe estar seleccionado. Tipos disponibles:
    • 📝 Resumen (HTML con h2, h3, p, ul, strong)
    • ❓ Quiz (10 preguntas de opción múltiple)
    • 🗂️ Flashcards (mínimo 8 tarjetas)
    • 🧠 Mapa conceptual (HTML con divs anidados y estilos inline)
  • Personalización por materia y tema específico
  • Campo opcional de notas adicionales del tutor/estudiante
  • Indicador de progreso con barra animada y mensajes de estado

2.4. Especificación Técnica de la API de Gemini NUEVO

Endpoint

POST https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent?key={API_KEY}

Estructura del Request Body

{
  "contents": [{
    "parts": [
      // Imágenes como inline_data
      { "inline_data": { "mime_type": "image/jpeg", "data": "{base64_string}" } },
      // ... más imágenes ...
      // Texto del prompt como último part
      { "text": "{instrucciones_del_prompt}" }
    ]
  }],
  "generationConfig": {
    "temperature": 0.7,
    "maxOutputTokens": 8000
  }
}

Estructura de Respuesta Esperada

// La API devuelve:
data.candidates[0].content.parts[0].text → string con JSON

// El JSON parseado debe tener esta estructura:
{
  "titulo": "Título del tema detectado",
  "resumen": "HTML string (si fue solicitado)",
  "quiz": [
    {
      "pregunta": "...",
      "opciones": ["A)...", "B)...", "C)...", "D)..."],
      "respuesta_correcta": 0,  // índice 0-3
      "explicacion": "..."
    }
  ],
  "flashcards": [
    { "frente": "Término", "reverso": "Definición" }
  ],
  "mapa_conceptual": "HTML string"
}

Post-procesamiento de la respuesta

  • Eliminar markdown fences: rawText.replace(/```json\s*/gi, '').replace(/```\s*/g, '').trim()
  • Parsear JSON con try/catch y mensaje amigable si falla

2.5. Prompt Engineering NUEVO

El prompt se construye dinámicamente con las siguientes variables:

  • Nombre del estudiante: personaliza las respuestas ("para {nombre}")
  • Nivel educativo: mapeado via gradeMap:
    ClaveDescripción enviada a IA
    primaria_baja"niño de 6 a 9 años, 1°-3° primaria"
    primaria_alta"niño de 9 a 12 años, 4°-6° primaria"
    secundaria"adolescente de 12 a 15 años, secundaria"
  • Materia y tema: contextualizan el análisis
  • Notas del tutor: instrucciones adicionales opcionales
  • Formato de salida: JSON puro sin markdown, con estructura específica según tipos seleccionados
  • Idioma: "Responde SIEMPRE en español"

2.6. Sistema de Evaluación (Quiz)

  • Cuestionarios interactivos con 4 opciones múltiples (A, B, C, D)
  • Retroalimentación inmediata:
    • Correcta: fondo verde + explicación
    • Incorrecta: opción marcada en rojo + correcta en verde + explicación
  • Barra de progreso visual por pregunta
  • Contador de respuestas correctas en tiempo real
  • Opciones se deshabilitan tras responder (.disabled)
  • Vista de resultados finales con:
    • Círculo de puntaje con sombra
    • Emoji contextual: 🏆 ≥90%, 🌟 ≥70%, 👍 ≥50%, 💪 <50%
    • Mensaje motivacional adaptado al porcentaje
    • Opciones: "Volver a intentar" y "Ver material"

2.7. Historial de Sesiones

  • Almacenamiento en localStorage bajo clave sb_history
  • Cada entrada contiene: { id, titulo, date, types, data }
  • Límite de 30 entradas (FIFO: la más antigua se elimina)
  • Formato de fecha: toLocaleDateString('es-CR', { day: '2-digit', month: 'short', year: 'numeric' })
  • Cargar sesión del historial re-renderiza los resultados y recarga el quiz si existe
  • Estado vacío con mensaje e icono cuando no hay historial

2.8. Interfaz de Usuario

  • Sistema de pestañas (Inicio, Generar, Practicar, Historial) con emojis
  • Diseño responsive con breakpoint en 600px:
    • Form rows pasan de 2 columnas a 1
    • Tabs reducen tamaño de fuente
    • Emojis de tabs se ocultan en móvil
  • Header sticky con logo y badge "✨ Con IA"
  • Contenedor con max-width 860px centrado
  • Animaciones fadeUp en elementos que aparecen

2.9. Sistema de Notificaciones Toast NUEVO

  • Elemento #sb-toast creado dinámicamente al primer uso
  • Posición: fixed, bottom 24px, centrado horizontalmente
  • Estilo: fondo oscuro (#1F2937), texto blanco, border-radius 20px
  • Auto-ocultamiento: fade-out a los 2800ms
  • Cancelación de toast previo si se muestra otro nuevo
  • Usado para: confirmaciones (✅), advertencias (⚠️), errores (❌), cámara (📸), copiado (📋)

2.10. Funciones Adicionales

  • Copiar texto: navigator.clipboard.writeText() con feedback toast
  • Flashcards interactivas: Click para flip (frente/reverso) con cambio de fondo
  • Impresión: Abre ventana nueva con estilos simplificados y ejecuta window.print()
  • Escape de HTML: Función escHtml() para prevenir XSS en contenido dinámico

5. Consideraciones de Seguridad NUEVO

⚠️ Riesgo conocido: La API Key de Gemini se envía directamente desde el frontend al endpoint de Google. Esto significa que la key es visible en las herramientas de desarrollo del navegador y en el código fuente.

5.1. Mitigaciones Actuales

  • La API Key se almacena solo en localStorage del navegador del usuario
  • No se envía a ningún servidor propio
  • Campo con type="password" para ocultarla visualmente
  • Cada usuario usa su propia API Key gratuita
  • Sanitización de HTML con escHtml() para prevenir XSS en contenido renderizado

5.2. Mejoras Recomendadas (Futuro)

  • Implementar un proxy backend ligero (Cloud Function / Edge Function) para ocultar la API Key
  • Agregar rate limiting en el cliente para evitar abuso
  • Validar el tamaño máximo de imágenes antes de enviar (recomendado: 4MB por imagen)
  • Implementar CSP (Content Security Policy) headers

6. Mejoras Recomendadas NUEVO

6.1. Accesibilidad (a11y)

  • El HTML ya usa role="tablist" en las pestañas — expandir con role="tab", aria-selected, aria-controls
  • Agregar aria-label a botones con solo emoji (captura, eliminar)
  • Implementar navegación por teclado en tabs y quiz options
  • Verificar contraste de colores con herramienta WCAG (especialmente --text3: #9CA3AF)
  • Agregar alt descriptivos a imágenes de preview

6.2. PWA / Modo Offline

  • Agregar manifest.json para instalabilidad
  • Implementar Service Worker para cacheo de assets y contenido generado
  • Permitir uso offline del historial y quiz (ya están en localStorage)

6.3. Optimización de Rendimiento

  • Comprimir imágenes antes de enviar a la API (canvas resize a max ~1500px de lado mayor)
  • Limitar número máximo de imágenes simultáneas (sugerido: 10)
  • Implementar lazy loading para el historial si crece
  • Limitar tamaño total de localStorage (sugerido: alertar si > 5MB)

6.4. UX Adicional

  • Confirmar antes de eliminar historial completo
  • Botón para eliminar entradas individuales del historial
  • Exportar/importar historial como JSON para respaldo
  • Modo oscuro (dark theme) usando las variables CSS existentes
  • Sonidos/vibraciones sutiles al responder quiz (opcional)
